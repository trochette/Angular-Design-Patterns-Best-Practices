Tip #2: Keep controllers simple

Keep your scope uncluttered

		
		[...]

		defineListeners:function(){
		   /**
			* Destroy function don't need to be called from template, 
			* thus no need to make it publicly available via scope.
			*/
			this.$scope.$on('$destroy',this.destroy.bind(this));
		},

		[...]

		defineScope:function(){
		   /**
			* Title needs to be bind and displayed by the view.
			* In that case we need to make it publicly available via scope.
			*/
			this.$scope.title = "I love OpenCode.ca"
		},

		[...]
Angular dirty check each properties when it thinks some values changed. Every $digest or $apply will trigger this check. Carefully selecting what is available on $scope and minimizing logic view bound functions is the key to build snappy applications.
